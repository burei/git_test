1. github에 레포지터리 생성
2. 로컬에 해당 레포지터리의 루트 폴더 생성
3. 루트 폴더에서 
   - git init
   - git remote add origin https://github.com/burei/git_test
4. 루트 폴더에 Sequence.txt 파일 생성 
5. git status 실행
   - No commits yet
   - Untracked files:
       (use "git add <file>..." to include in what will be committed)
	Sequence.txt
   - nothing added to commit but untracked files present (use "git add" to track)
6. git commit -m "Hello git_test"
7. git push origin master

- Github 사이트에서 파일을 수정하거나 추가하는 경우
1. 수정 및 추가
2. 로컬 루트 폴어데서 pull 해야할 듯.
   - git pull origin master

- 로컬에서 dev 브런치 생성후 체크아웃해서 master -> dev로 브랜치 이동
1. git branch dev
2. git checkout dev

- 로컬에서 새로운 브런치 생성
1. git branch stream_command
2. git checkout stream_command
3. git push origin stream_command
